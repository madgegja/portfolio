CS AI Chatbot

Claude API + FastAPI + RAG-based intelligent CS chatbot. Hybrid 90/10 response model (90% vetted templates + 10% AI glue) with fraud detection, 3-tier smart escalation, and multilingual support across 10 markets and 10 languages.

! CHALLENGE

~1,650 CS tickets/month across 10 markets, with 38% being repetitive self-serviceable queries (password resets, subscription cancellations, payment inquiries). Agents spent most of their time on routine responses instead of complex cases. No fraud detection existed at the CS entry point — malicious refund patterns were only caught after costly escalation. Third-party cancellation services (Minna/Ethoca) were generating unactionable tickets.

▶ STRATEGY

Built a hybrid AI chatbot with a 90/10 response model — 90% manual-vetted templates for accuracy + 10% AI natural language for conversational flow. Designed a multi-stage pipeline: Step 0.5 (third-party block) → Language Detection → Fraud Scoring (9 Red Flags, 0~100) → Intent Classification (7 categories × 32 sub-intents) → 3-tier Routing (L1 immediate escalation / L2 AI draft + agent review / L3 full self-service). Response quality scoring (0.0~1.0) ensures output reliability.

✓ RESULTS

  • 38% self-service target (~630 tickets/month automated)
  • 161 test cases across 5 test files — unit + integration
  • 30 multilingual templates (3 categories × 10 markets)
  • 9-indicator fraud scoring with boundary hardening (49/50 clamp)
  • CI/CD pipeline: test coverage → Docker push (GHCR) → staging deploy

CS Analysis & Reporting System

End-to-end CS data analysis pipeline — from raw Zendesk exports to structured weekly/monthly reports with charts. Standardized category taxonomy across 10 global markets with multilingual VOC analysis.

! CHALLENGE

Raw Zendesk export data contained massive noise — auto-responses, spam, merged tickets, system messages made up ~75% of records. No standardized category taxonomy existed across markets, making cross-market comparison impossible. Manual Excel-based reporting was error-prone, slow, and couldn't scale across 10 markets and 11 languages.

▶ STRATEGY

Built a Python pipeline with 3-stage filtering (auto-response / spam / merge removal) → 4-level priority category classification engine → automated report generation with Chart.js visualizations. Defined a unified category taxonomy applicable across all 10 markets. Implemented solved_at (KST) date normalization and Saturday–Friday weekly cycles to align with operational reality.

✓ RESULTS

  • 12,920 raw → 3,227 cleaned tickets (75% noise reduction)
  • 4 report types auto-generated (weekly detail + executive + monthly + VOC)
  • Unified category taxonomy adopted across all 10 markets
  • Cross-market comparison enabled for the first time

BizOps Artifacts Portal

Centralized public documentation portal hosting 29+ operational artifacts — CS manuals (EN/KO), chargeback dispute templates, AI chatbot planning suite (13 docs), analysis reports, and weekly report archives. Single source of truth for team operations.

! CHALLENGE

29+ operational documents (CS manuals, chargeback guides, chatbot specs, analysis reports) were scattered across Confluence, Google Drive, and local files. No single source of truth for operational reference existed. New team members had no centralized onboarding resource, and finding the latest version of any document required asking multiple people.

▶ STRATEGY

Built a static documentation portal on GitHub Pages with categorized navigation (7 categories), accordion-based UI, and stat-card overview. All 29+ documents authored solo — leveraging Claude AI for content generation while applying domain expertise for accuracy. Automated deployment via GitHub Actions (push to main triggers sync). Designed print-optimized layouts for PDF conversion.

✓ RESULTS

  • 29+ documents across 7 categories, bilingual (EN/KO)
  • Single URL for all operational artifacts — eliminated document hunting
  • Adopted as official team onboarding resource
  • Auto-deploy pipeline: git push → GitHub Actions → live portal
